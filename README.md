# Тестовое Задание На Шахматной Доске

## Задание

- На шахматной доске находятся в произвольной позиции N ладей (4-6).
- Они все одновременно начинают ходить на случайные позиции (при этом перемещаться они, естественно, могут только горизонтально либо вертикально; ход ладьи атомарен - фигура перемещается из начальной позиции в конечную за один шаг).
- Между каждыми ходами каждая фигура делает паузу 200-300 миллисекунд.
- Если на пути фигуры оказывается другая, она ждет, пока путь освободится.
- Если в течение 5 секунд проход не освободился, выбирается другая позиция аналогичным случайным образом.
- Всё заканчивается, когда все фигуры сделают по 50 ходов.
- GUI никакое не требуется, достаточно будет понятного текстового вывода.

## Критерии

- Оценивается корректность реализации условий.
- Если выполняется многопоточно, то проверяется насколько качественно сделана работа с потоками.
- Если выполняется однопоточно в виде асинхронной модели, то проверяется качество асинхронности.
- Проверяется дизайн решения: насколько читаем код, насколько прозрачно код отражает предметную область.
- Не стоит пытаться удивить экзотическими решениями. стоит выполнить задание так, если бы это была задача на текущем месте работы.
- Решение должно собираться под Линукс, кроме исходников должен быть сборочный скрипт.
- Задание в среднем занимает 2 - 3 часа.

## Design

### Подход

Многопоточная реализация на C++20;
Каждая ладья работает в отдельном потоке;
Доска — разделяемый ресурс, защищённый мьютексом;

### Классы

**Board** — 8x8 сетка клеток, хранит указатели на ладьи или nullptr;
**Rook** — позиция (col, row), счётчик ходов, идентификатор;

### Алгоритм хода ладьи

- Выбрать случайную целевую клетку (та же строка или столбец);
- Проверить, свободен ли путь до цели (захватить мьютекс);
- Если путь занят — ждать через `condition_variable` до 5 секунд;
- Если 5 секунд прошло — выбрать новую случайную цель, повторить;
- Если путь свободен — переместить ладью атомарно, освободить мьютекс;
- Уведомить остальные потоки через `condition_variable::notify_all`;
- Сделать паузу 200–300 мс, повторить до 50 ходов;

### Синхронизация

`std::mutex` защищает состояние доски при чтении и записи;
`std::condition_variable` будит ожидающие ладьи после каждого хода;
`std::jthread` заменяет `std::thread` - авто-join при уничтожении;
`std::latch` синхронизирует одновременный старт всех ладей;

### Текстовый вывод

Каждый ход печатает: ID ладьи, старую позицию, новую позицию, счётчик;
Вывод синхронизирован отдельным `std::mutex` для читаемости;

### Допы от C++20

- `std::jthread` — поток с авто-join и встроенным `stop_token`;
- `std::stop_token` — кооперативная отмена потока без флагов вручную;
- `std::latch` — одноразовый барьер для синхронного старта ладей;
- `std::atomic::wait/notify_all` — лёгкая альтернатива `condition_variable`;

### Стандартные библиотеки

- `<thread>` - `std::jthread` для потока каждой ладьи;
- `<mutex>` - `std::mutex`, `std::unique_lock` для доски и вывода;
- `<condition_variable>` - ожидание освобождения пути;
- `<atomic>` - атомарный счётчик, `wait/notify_all`;
- `<latch>` - барьер синхронного старта;
- `<random>` - `std::mt19937` для случайных позиций;
- `<chrono>` - паузы 200–300 мс и таймаут 5 секунд;
- `<vector>` - хранение ладей и потоков;
- `<array>` - фиксированная сетка доски 8x8;

### Структура файлов

```
sources/
    main.cpp - точка входа, инициализация, запуск потоков;
    board.hpp - класс Board, доступ к клеткам;
    rook.hpp - класс Rook, логика хода;
scripts/
    run.sh - построить на cmake и запустить
CMakeLists.txt - билд система, скрипт постройки
build/ - все игнорированные сгенерированные файлы
```
